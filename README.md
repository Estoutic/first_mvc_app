# first_mvc_app

Учебный проект — простое Java веб-приложение с многомодульной структурой на Maven.

## Что здесь сделано

Показан базовый MVC-подход без фреймворков: разделение кода на слои через отдельные Maven-модули.

## Структура

```
database/   — модель данных (User) и доступ к БД (UserDao)
service/    — бизнес-логика (UserService) и DTO (UserDto)
web/        — сервлет (UserServlet), принимает HTTP-запросы и отдаёт HTML
```

## Как работает

1. Браузер делает GET-запрос на `/user`
2. `UserServlet` вызывает `UserService.getUser()`
3. `UserService` обращается к `UserDao`, получает `User`, конвертирует в `UserDto`
4. Сервлет рендерит простую HTML-страницу с именем пользователя

Сейчас `UserDao` возвращает захардкоженного пользователя `John Doe` — реальное подключение к PostgreSQL не реализовано (зависимость уже добавлена в pom.xml).

## Технологии

- Java 21
- Jakarta Servlet API 6.1
- Maven (multi-module)
- PostgreSQL (драйвер подключён, интеграция не реализована)

## Сборка и запуск

```bash
mvn clean package
```

Задеплоить `web/target/web.war` на любой совместимый сервлет-контейнер (например, Tomcat 10+).
